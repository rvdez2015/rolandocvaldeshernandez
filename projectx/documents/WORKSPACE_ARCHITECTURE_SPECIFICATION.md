# Project X Workspace Architecture Specification

## Principle
A teacher should never need to leave the Lesson Workspace to prepare, teach, assess, review or improve a lesson.

## Standard Workspace Layout
1. Breadcrumb navigation
2. Workspace header
3. Teaching ribbon
4. Context panel
5. Workspace tabs
6. Active content area
7. Status/readiness bar

## Initial Workspace Types
- Curriculum Intelligence Workspace
- Unit Workspace
- Lesson Workspace / Digital Lesson Twin 2.0
- Professional Standards Workspace
- Development Studio Workspace

## Core Tabs
- Overview
- Objectives
- Resources
- Assessment
- Evidence
- Reflection
- History
- AI Assistant
- Settings

## Design Rule
Every workspace must declare its purpose, components, data sources, events, quick actions, dependencies and validation checks in Development Studio before implementation.
